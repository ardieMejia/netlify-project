Command lines I keep forgetting

That -print0 | xargs -0 part always gets me. This command is so important, this should be in permanent memory by now, but its not! (confused face).

Nowadays, I came up with a strategy inside my head, that the key to learning new Linux commands once you get used to them, is to look for a ‘dominating’ tool in a single line. In the example above, xargs is the dominating tool. So its best to not get bamboozled, and realize all that -print0 | xargs -0 is just xargs fluff.

This wget is not that important for me, but every time I do need it, its never anywhere on my local system but online.

  • -a: Archive mode. Plenty of explanation online. To put it simple, this is perfect for backups (mirrors everything: timestamps, permissions etc..) AND makes future backups easier (Eg: updates instead of redundant copies).
  • -v: verbose
  • -z: compress during transfer (not important)

      • Further Notes:

    • When using pattern-matching, rsync include/exclude behaviour can be weird. This stackoverflow advice explains well, but adds that it is quirky and not intuitive.
    • Forget reading into everything here! Instead, when using both, always put --include= before --exclude by habit.
    • And since rsync includes files by default, so:
      • When you want to backup everything except certain filetypes: --exclude only
      • When you want to exclude everything except certain filetypes: --include first, then --exclude
